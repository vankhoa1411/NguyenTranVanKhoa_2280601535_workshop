### Week 6 Objectives:

* Perform advanced hands-on practice with Amazon EC2.
* Understand how Security Groups work and learn best practices for securing AWS servers.
* Practice connecting to Amazon EC2 instances via SSH.
* Deploy a simple web application on an Amazon EC2 instance.

### Tasks to Be Completed This Week:

| Day | Task | Start Date | Completion Date | Reference |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ | --------------- | ----------------------------------------- |
| Monday | - Review Amazon EC2 concepts <br> - Launch a new Amazon EC2 instance for practice <br> - Select the Amazon Linux 2 AMI and configure an instance type compatible with the AWS Free Tier | 25/05/2026 | 25/05/2026 | <https://docs.aws.amazon.com/ec2/> |
| Tuesday | - Learn and configure **Security Groups** <br>&emsp; + Open SSH port (22) <br>&emsp; + Open HTTP port (80) <br>&emsp; + Open HTTPS port (443) <br>&emsp; + Configure inbound and outbound rules | 26/05/2026 | 27/05/2026 | <https://docs.aws.amazon.com/vpc/latest/userguide/VPC_SecurityGroups.html> |
| Wednesday | - Practice connecting to an EC2 instance via SSH using Terminal/PowerShell <br>&emsp; + Use a Key Pair (.pem) <br>&emsp; + Verify the instance status <br>&emsp; + Execute basic Linux commands | 28/05/2026 | 28/05/2026 | <https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connect-to-linux-instance.html> |

### Week 6 Achievements:

* Understood the process of launching and configuring an **Amazon EC2** instance for application deployment.

* Gained a clear understanding of the role of **Security Groups** in controlling inbound and outbound traffic for Amazon EC2 instances.

* Successfully configured the required service ports, including **SSH (22)**, **HTTP (80)**, and **HTTPS (443)**.

* Successfully connected to an Amazon EC2 instance using **SSH** with a **Key Pair**.

* Became familiar with basic Linux system administration tasks, including updating packages, creating directories, and managing files.

* Successfully installed and configured a web server (**Apache** or **Nginx**) on an Amazon EC2 instance.

* Successfully deployed a simple HTML website and accessed it through the EC2 public IP address.

* Learned how to monitor the status of an EC2 instance and troubleshoot common issues related to SSH connections and website accessibility.

* Completed deployment documentation for Amazon EC2 and the web server, providing a reference for building more advanced cloud-based systems in the following weeks.

* Improved Linux server administration skills and practical experience in deploying applications on the AWS Cloud platform.
