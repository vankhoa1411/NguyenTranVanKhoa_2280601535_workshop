### Week 8 Objectives:

* Learn about **AWS Identity and Access Management (IAM)**.
* Understand authentication and access control mechanisms on AWS.
* Practice creating IAM Users, Groups, Roles, and Policies.
* Apply fundamental security best practices when managing AWS accounts.

### Week 8 Achievements:

* Gained a comprehensive understanding of the role of **AWS Identity and Access Management (IAM)** in managing identities and controlling access to AWS resources.

* Learned the differences between the core IAM components:
  * IAM User
  * IAM Group
  * IAM Role
  * IAM Policy

* Successfully created multiple IAM Users for different account management purposes.

* Learned how to create and manage IAM Groups to assign permissions to multiple users efficiently.

* Understood how to use **Managed Policies** and **Custom Policies** to control access to AWS resources.

* Successfully created IAM Roles and attached them to Amazon EC2 instances, enabling secure access to AWS services without using Access Keys.

* Applied the **Principle of Least Privilege** by granting only the permissions required for specific tasks, improving overall system security.

* Became familiar with enabling **Multi-Factor Authentication (MFA)** to enhance AWS account security.

* Completed comprehensive documentation on AWS user management and access control, providing a solid foundation for building secure Serverless applications in the following weeks.

* Improved awareness of cloud security best practices and effective identity and access management in AWS.
