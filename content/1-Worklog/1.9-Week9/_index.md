### Week 9 Objectives:

* Learn the fundamentals of AWS Networking and Amazon Virtual Private Cloud (VPC).
* Understand how to build a network infrastructure on the AWS platform.
* Practice creating a VPC, Subnets, and an Internet Gateway.
* Understand how AWS services communicate within the same cloud environment.

### Week 9 Achievements:

* Gained a clear understanding of the role of **Amazon VPC** in building secure and isolated network infrastructures on AWS.

* Learned the concept of **CIDR Blocks** and how IP address ranges are allocated within a virtual network.

* Successfully created an Amazon VPC for deploying AWS resources.

* Understood the differences between **Public Subnets** and **Private Subnets**, as well as their common use cases.

* Successfully configured an **Internet Gateway** and **Route Tables** to enable Internet access for resources in the Public Subnet.

* Understood the roles of **Security Groups** and **Network Access Control Lists (NACLs)** in protecting AWS resources.

* Learned how to design a basic network architecture for cloud-based applications.

* Understood the communication flow between services such as **Amazon EC2**, **Amazon S3**, **Amazon RDS**, and the Internet within an AWS environment.

* Completed a VPC network architecture diagram and documented the entire configuration process for future reference during project design.

* Acquired sufficient networking knowledge to support the implementation of the **Cloud-Native AI Document Extractor & Analytics** project using a Serverless architecture.
