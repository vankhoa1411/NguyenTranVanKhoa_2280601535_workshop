#### Overview

In this step, we will build the **Admin Notifications** feature for the **DocuMind AI** system.

Admin Notifications enable administrators to monitor critical system events such as document processing failures, OCR extraction issues, AI provider errors, messages routed to the Dead Letter Queue (DLQ), abnormal user upload rates, or security threats.

Unlike User Notifications which relate solely to individual user documents, Admin Notifications are focused on system-wide monitoring.

---

#### Objectives of This Step

Upon completing this step, you will:

* Create notifications targeted at Administrators.
* Store admin notifications in PostgreSQL.
* Render notifications within the Admin Panel.
* Categorize notifications by severity level.
* Generate alerts for background worker errors.
* Generate alerts when an AI provider fails or fallbacks are triggered.
* Generate alerts for notable security actions.
* Prepare data for system monitoring and audit logs.

---

#### The Role of Admin Notifications

Admin Notifications are used to monitor:

* Document processing failures.
* Textract OCR failures.
* Gemini/OpenAI provider failures.
* AI provider fallback triggers.
* SQS messages routed to the DLQ.
* Multiple invalid file upload attempts.
* Abnormal user activity.
* Database connection drops.
* AWS service or IAM policy configuration issues.
* Security alarms generated by the system.

Examples:

```text id="wir8ds"
AI provider fallback used.
Document processing failed.
SQS message moved to DLQ.
Multiple failed uploads detected.
```

#### Notification Model for Administrators

Admin notifications use the same `Notification` model, but route through the `roleTarget` field.

Example:

```json id="n1lylo"
{
  "roleTarget": "ADMIN",
  "type": "AI_PROVIDER_FALLBACK",
  "severity": "warning",
  "title": "AI provider fallback used",
  "message": "OpenAI failed. The system used Gemini as fallback provider.",
  "metadata": {
    "primaryProvider": "openai",
    "fallbackProvider": "gemini",
    "documentId": "doc_001"
  },
  "isRead": false
}
```

#### Severity Levels

Admin Notifications categorize issues by urgency:

| Severity   | Description                        |
| ---------- | ---------------------------------- |
| `info`     | Normal operational updates         |
| `success`  | Task completed successfully        |
| `warning`  | Non-fatal issues requiring attention|
| `error`    | Actionable execution errors        |
| `critical` | Critical errors requiring immediate intervention |

Examples:

```text id="evoyf3"
info
warning
error
critical
```

#### Admin Notification Service

Create the function:

```ts id="q9l8h0"
import { prisma } from "../prisma/client";
 
export async function createAdminNotification(params: {
  type: string;
  severity: string;
  title: string;
  message: string;
  metadata?: any;
}) {
  return prisma.notification.create({
    data: {
      roleTarget: "ADMIN",
      type: params.type,
      severity: params.severity,
      title: params.title,
      message: params.message,
      metadata: params.metadata || {},
      isRead: false
    }
  });
}
```

#### Recommended Admin Notification APIs

Endpoints exposed to administrators:

| API                                         | Purpose                          |
| ------------------------------------------- | -------------------------------- |
| `GET /api/admin/notifications`              | Fetch Admin notifications list   |
| `GET /api/admin/notifications/unread-count` | Get unread alerts count          |
| `PUT /api/admin/notifications/:id/read`     | Mark a notification as read      |
| `PUT /api/admin/notifications/read-all`     | Mark all notifications as read   |
| `POST /api/admin/notifications/broadcast`   | Broadcast alerts to system users |
| `DELETE /api/admin/notifications/:id`       | Delete a notification record     |

All administrative endpoints must invoke:

```text id="fj4xyp"
requireAuth
requireAdmin
```

#### Troubleshoot Scenarios

| Error                                | Cause                                    | Solution                                  |
| ------------------------------------ | ---------------------------------------- | ----------------------------------------- |
| Non-admin users access alerts        | Missing `requireAdmin` validation check  | Check backend route middleware rules      |
| Admin alerts are missing             | Query filters missing `roleTarget=ADMIN` | Verify admin notification API endpoints   |
| Incorrect status color codes         | Frontend styling mapping error           | Check Tailwind UI badge component styles  |
| Metadata missing `documentId`        | Service failed to pass metadata payload  | Verify parameters passed to notification service|
| Broadcast alerts fail to send        | Incorrect recipient query logic          | Check database user query filters         |
| Unread badge count is incorrect      | Query did not filter by `isRead=false`   | Verify unread API query parameters        |
