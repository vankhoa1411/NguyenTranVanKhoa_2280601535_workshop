#### Build Frontend Dashboard

In this section, we will build the **Frontend Dashboard** for the **DocuMind AI** system.

The Frontend Dashboard is the primary user interface enabling users to interact with the system. Users can log in, upload documents, track document processing progress, view OCR extraction results, inspect AI Analysis summaries, chat with documents, and experiment with advanced features in the Enterprise Sandbox.

Once backend features such as the upload API, SQS Worker, Textract OCR, AI Gateway, and PostgreSQL + Prisma database are established, the frontend serves to display the entire document processing pipeline in a visual, user-friendly format.

---

#### The Role of Frontend in DocuMind AI

In the **DocuMind AI** project, the frontend is responsible for:

* Rendering the Home Page describing the system.
* Enabling user registration and login flows.
* Displaying user authentication state, avatars, and profile details.
* Supporting PDF, PNG, JPG, and JPEG document uploads.
* Listing all uploaded documents.
* Monitoring document processing status.
* Displaying OCR Result text extracted via Amazon Textract.
* Displaying AI Analysis summaries generated by Gemini/OpenAI.
* Supporting AI Chat/RAG interactive questions.
* Providing the Enterprise Sandbox to test semantic search, entities, and analytics.
* Enforcing conditional UI rendering based on User and Admin roles.

#### Displaying Document Statuses on the Dashboard

The frontend must visually represent document processing stages:

```text
PENDING
UPLOADED
QUEUED
PROCESSING
OCR_COMPLETED
AI_ANALYZING
COMPLETED
FAILED
```

Progress tracking definitions:

| Status          | Progress | Description          |
| --------------- | -------: | -------------------- |
| `PENDING`       |        5 | Initial metadata set |
| `UPLOADED`      |       15 | File uploaded to S3  |
| `QUEUED`        |       25 | Waiting for worker   |
| `PROCESSING`    |       45 | OCR extraction running|
| `OCR_COMPLETED` |       60 | OCR finished         |
| `AI_ANALYZING`  |       75 | AI analysis running  |
| `COMPLETED`     |      100 | Completed            |
| `FAILED`        |        0 | Processing error     |

#### Backend APIs Consumed

Core APIs called by the frontend:

| API                                    | Purpose                                      |
| -------------------------------------- | -------------------------------------------- |
| `POST /api/auth/register`              | User registration                            |
| `POST /api/auth/login`                 | User authentication                          |
| `GET /api/auth/me`                     | Fetches authenticated profile details        |
| `POST /api/auth/logout`                | Session destruction                          |
| `POST /api/documents/upload`           | Uploads document payload                     |
| `GET /api/documents`                   | Lists user-owned documents                   |
| `GET /api/documents/:id`               | Fetches details of a specific document       |
| `GET /api/documents/:id/status`        | Queries document pipeline stage              |
| `GET /api/documents/:id/ocr`           | Fetches document OCR text                    |
| `GET /api/documents/:id/analysis`      | Fetches document AI analysis summaries       |
| `POST /api/ai/chat`                    | Submits context-aware Q&A queries            |
| `POST /api/enterprise/semantic-search` | Performs semantic search queries in Sandbox  |

#### Authentication State Management

The frontend tracks the following user parameters:

```text
user
token
isAuthenticated
role
loading
```

Upon a successful login session:

* Store the session token.
* Call the `/api/auth/me` profile endpoint.
* Update avatar and name displays.
* Hide Login/Register options.
* Enable dashboard navigation.
* Render the Admin Panel menu option if the role matches `ADMIN`.

---

#### Document Dashboard State

The dashboard tracks the following document parameters:

```text
documents
selectedDocument
uploading
processingStatus
loading
error
```

Upon a successful upload:

* Append the new document object to the list.
* Initialize status state to `QUEUED`.
* Initiate background polling queries.
* Trigger a successful upload notification.

---

#### AI Chat/RAG State

The AI Chat UI tracks:

```text
messages
question
provider
loading
sources
confidence
```

Upon submitting a prompt:

* Send the `documentId`, `question`, and `provider` to the backend.
* The backend retrieves the document OCR text context.
* The AI Gateway routes the request to Gemini or OpenAI.
* The frontend displays the returned answer.
* The session history is logged in the database.

#### Client-side Security Safeguards

Ensure the following frontend constraints are maintained:

* Do not expose Gemini/OpenAI API keys on the client-side.
* Do not expose AWS secret credentials.
* Only pass JWT tokens in request header authorization blocks.
* Restrict page routing for documents belonging to other users.
* Implement login protected routes for dashboard views.
* Validate the `ADMIN` role state before rendering admin paths.
* Obfuscate verbose system errors from end-users.

#### Loading, Empty, and Error States

The frontend should have clear UI placeholders:

| State        | Description                               |
| ------------ | ----------------------------------------- |
| Loading      | Retrieving database details               |
| Empty        | No documents available                    |
| Uploading    | Transmitting document payload             |
| Processing   | Document is currently processing          |
| Completed    | Document finished processing              |
| Failed       | Document processing failure               |
| Unauthorized | User is not logged in                     |

Example empty state text:

No documents yet.
Upload your first document to start extracting and analyzing content with AI.

#### Common Troubleshoot Scenarios

| Error                                | Cause                                    | Solution                                  |
| ------------------------------------ | ---------------------------------------- | ----------------------------------------- |
| Login succeeds but UI does not update| Auth state not updating                  | Validate AuthContext or Zustand variables |
| S3 uploads fail with CORS error      | Incorrect backend or S3 CORS setup       | Check bucket CORS rules                   |
| Dashboard status fails to refresh    | Polling inactive or worker stopped       | Check Document Status API                 |
| AI Analysis results are empty        | Document not in `COMPLETED` state        | Verify worker and AI Gateway status       |
| 401 Unauthorized                     | Expired session token                    | Re-authenticate the user session          |
| 403 Forbidden                        | Attempting to access other user's files  | Check backend query filters               |
| Broken layouts on mobile views       | Mobile layout not optimized              | Verify Tailwind breakpoint utilities      |
| Bedrock reference text found         | Hardcoded legacy text                    | Replace Bedrock references in code        |
